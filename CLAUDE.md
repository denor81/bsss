## BSSS - Bash System Security Suite

BSSS is a modular Bash framework for Ubuntu system security management. This is NOT a collection of simple scripts—it's a pipeline-first architecture built on Unix philosophy.

## Development Commands

**Running the system:**
```bash
sudo ./local-runner.sh  # Main entry point (requires root)
./local-runner.sh -h     # Show help
./local-runner.sh -u     # Uninstall
```

**Building/distribution:**
```bash
./build-archive.sh       # Create distribution archive
./oneline-runner.sh      # One-liner installer/downloader
```

**Function management:**
```bash
./generate_function_map.sh  # Regenerate function_map.txt after adding new functions
```

## Architecture Overview

### Pipeline-First Design

The core philosophy: data flows through pipes using NUL-separators (`\0`), not variables.

```
"NUL lives in the pipe, but dies in the variable."
```

- **FD 1 (stdout)**: Pure data only for the next pipeline stage
- **FD 2 (stderr)**: All UI, logging, diagnostics, user interaction
- Use `xargs -0`, `sort -z`, `mapfile -d ''`, `read -d ''` for NUL-delimited streams

### Module System

Modules are numbered bash scripts in `modules/` directory:

```
modules/
├── 01-os-check.sh          # Check: OS compatibility validation
├── 02-update-system.sh     # Modify: System updates
├── 03-sys-reload-check.sh  # Check: Reboot required detection
├── 04-ssh-port-*.sh        # Modify: SSH port management
├── 05-ufw-*.sh             # Modify: UFW firewall rules
└── common-helpers.sh       # Shared utilities
```

**Module types** (defined in module headers):
- `MODULE_TYPE_CHECK`: Validates system state (e.g., OS compatibility)
- `MODULE_TYPE_MODIFY`: Performs system modifications

**Module discovery**: Uses glob pattern `[0-9][0-9]*.sh` for automatic discovery.

### Role-Based Namespacing

All functions MUST use prefixes to indicate responsibility:

| Prefix    | Role                           |
|-----------|--------------------------------|
| `orchestrator::` | High-level logic and scenario control |
| `ssh::`, `ufw::`, `grub::` | Service-specific providers |
| `io::`    | User interaction and logging   |
| `sys::`   | Low-level system checks        |

### Function Contracts

Every function MUST have annotations before its definition:

```bash
# @type:        Source | Filter | Transformer | Orchestrator | Sink
# @description: Brief description of what it does
# @stdin:       Input format (e.g., NUL-separated paths)
# @stdout:      Output format
# @exit_code:   0 - success
#               2 - user cancellation (SIGINT-like)
#               3 - system rollback (module terminated by rollback mechanism)
#               >0 - error (except 2, 3)
```

**Function types:**
- **Source**: Generates data stream (e.g., `get_paths`)
- **Filter/Transformer**: Consumes stdin, produces stdout (prefer awk, sed, grep, xargs)
- **Orchestrator**: Makes decisions, manages logic flow
- **Sink**: Consumes data without passing through

### Exit Code Standards

- `0`: Success
- `2`: User cancellation (interpret as informational, NOT an error)
- `3`: System rollback (module terminated by automatic rollback mechanism)
- `>0` (except 2, 3): Critical error

### Directory Structure

All scripts use `PROJECT_ROOT` as the base directory for path resolution:

**For scripts in project root** (local-runner.sh, bsss-main.sh):
```bash
readonly PROJECT_ROOT="$(cd "$(dirname "$(readlink -f "${BASH_SOURCE[0]}")" )" && pwd)"
```

**For scripts in subdirectories** (modules/*.sh, utils/*.sh):
```bash
readonly PROJECT_ROOT="$(cd "$(dirname "$(readlink -f "${BASH_SOURCE[0]}")" )" && pwd)/.."
```

**Path usage from PROJECT_ROOT**:
- Libraries: `${PROJECT_ROOT}/lib/vars.conf`
- Modules: `${PROJECT_ROOT}/modules/common-helpers.sh`
- Utils: `${PROJECT_ROOT}/utils/rollback.sh`

### Safety Principles

1. **NO `eval`** - Strictly forbidden. Use arrays, xargs, or direct calls.
2. **Idempotency** - Re-runs must not break the system. Use `# Generated by BSSS` markers.
3. **Atomic modifications** - Each significant change should be atomic.
4. **Automatic rollback** - Critical changes (SSH) use background watchdogs with SIGUSR1/SIGUSR2 signals.
5. **Root required** - All modifications require root privileges.

## Core Libraries

Located in `lib/`:

- **vars.conf**: All configuration constants (paths, timeouts, module types)
- **logging.sh**: Unified logging (`log_info`, `log_error`, `log::draw_border`)
- **user_confirmation.sh**: User interaction (`io::confirm_action`, `io::ask_value`)
- **uninstall_functions.sh**: Cleanup functionality

## Function Map

`function_map.txt` contains a complete registry of all functions.

Format: `ID | function_name() | type | description | file_path`

**Before writing new functionality:**
1. Check `function_map.txt` for existing solutions
2. Search by keywords in description or by prefixes (ssh::, ufw::, sys::)
3. Prefer composing existing functions via pipelines over duplicating code

**Regenerate after adding new functions:**
```bash
./generate_function_map.sh
```

## Helper Placement

- **Common helpers** (used in unrelated modules) → `modules/common-helpers.sh`
- **Domain-specific helpers** (used only in one module type) → co-locate with the module
  - SSH helpers → `modules/04-ssh-port-helpers.sh`
  - UFW helpers → `modules/05-ufw-helpers.sh`

## Pipeline Examples

**Orchestrator (high-level logic):**
```bash
orchestrator::bsss_config_not_exists() {
    ssh::ask_new_port | ssh::reset_and_pass | ufw::reset_and_pass | ssh::install_new_port
    orchestrator::actions_after_port_change
}
```

**Filter (pass-through with side effect):**
```bash
ufw::reset_and_pass() {
    local port=""
    [[ ! -t 0 ]] && IFS= read -r -d '' port || true  # Read from stdin if available
    ufw::delete_all_bsss_rules
    [[ -n "$port" ]] && printf '%s\0' "$port" || true  # Pass through if present
}
```

**Source (generates data):**
```bash
ssh::generate_free_random_port() {
    while IFS= read -r port || break; do
        if ! ssh::is_port_busy "$port"; then
            printf '%s\n' "$port"
            return
        fi
    done < <(shuf -i 10000-65535)
}
```

## Key Technical Details

- **Bash 5+** required
- **GNU Coreutils** for text processing (awk, sed, grep, sort, xargs)
- **ss** command (modern netstat replacement) for socket statistics
- **UFW** for firewall management
- **OpenSSH** for SSH configuration
- **NUL separators** (`\0`) for all data streaming
- **Modern standards**: `mapfile -d ''`, `read -d ''`, `sort -z`, `awk ORS="\0"`
